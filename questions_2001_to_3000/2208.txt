Problem Title: Minimum Operations to Halve Array Sum
Brute Force Method: Repeated Linear ScanLinearly search the array for the maximum element, halve it, subtract from sum. Repeat this entire process until the sum is halved.
BF Complexity Visuals: Repeated Array Scans
How to Draw BF: Draw the array. Draw a magnifying glass finding the biggest number. Erase it, write the half-value. Redraw the magnifying glass scanning the whole array from scratch.
Optimised Pattern: Max Heap (Priority Queue)
Opt Vis. Technique: Binary Tree (Heap Structure)
Dry Run (Opt): Push all elements into a Max Heap. Pop the root (the max), halve it, subtract from target sum, push it back. Repeat until target sum is reached.
How to Draw (Opt): Draw a tree where the parent is always larger than its children. Circle the root, draw an arrow pulling it out, halve it, and draw it trickling back down the tree structure.
Opt Complexity Visuals: Heap Operation Cycles
How to Draw Opt Complexity: Draw an $O(N)$ box for the initial heapify. Then draw a circular arrow labeled `K` times. Inside the loop, write `O(log N)` for extraction/insertion, showing $O(N + K \log N)$ total.
Mem Vis (BF): Mutating the original flat array in place (O(1) extra space).
Mem Vis (Opt): A balanced binary tree structure in memory representing the Priority Queue array (O(N) space).
