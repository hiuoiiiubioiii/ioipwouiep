Problem Title: Decremental String Concatenation
Brute Force Method: Recursion: For each new word, try appending it to the front OR the back of the current joined string. Calculate the final length for all 2^N paths.
BF Complexity Visuals: Exponential String Growth Tree
How to Draw BF: Draw a tree where each level doubles. Each node represents a fully concatenated string. With N=1000, this is 2^1000 strings.
Optimised Pattern: Massive String Heap (Visualizing huge strings being copied and stored at every level of recursion).
Opt Vis. Technique: Dynamic Programming (State: first_char, last_char, index)
Dry Run (Opt): Boundary-Only DP Table
How to Draw (Opt): Key insight: Only the first and last characters of the current string matter for future merges. State: dp[i][first][last] is the min length using words up to i with specific start/end chars. Transition by checking if word's edge matches string's edge.
Opt Complexity Visuals: Draw a 3D grid: 1000 (Index) x 26 (First Char) x 26 (Last Char). Fill the grid level by level using the previous level's results.
How to Draw Opt Complexity: $O(N \cdot 26 \cdot 26)$ Linear Bound
Mem Vis (BF): Draw a rectangle representing N x 676. Label it $O(N)$ since 676 is a constant overhead.
Mem Vis (Opt): Compact DP Array (Draw a 2D array 26x26 that is overwritten for each new word to save space).
