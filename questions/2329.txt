Problem Title: Product Sales Analysis V (Premium SQL)
Brute Force Method: (Conceptual) For every user, run a separate subquery scanning the entire sales table to calculate their total spending, then sort the final results.
BF Complexity Visuals: N-Scans over Database
How to Draw BF: Draw a table. Draw an arrow scanning the entire table from top to bottom, then repeat this full scan for every unique user.
Optimised Pattern: GROUP BY + Aggregation (SUM)
Opt Vis. Technique: Data Bucketing and Weighing
Dry Run (Opt): Join the Sales and Product tables. Group the results by `user_id`. Select the `user_id` and the `SUM(quantity * price)` as total spending. Order the results descending by total spending.
How to Draw (Opt): Draw a messy pile of receipts. Draw boxes labeled with user names. Sort the receipts into the boxes. Write the total sum on the outside of each box, then line the boxes up from heaviest to lightest.
Opt Complexity Visuals: Single Aggregation Pipeline
How to Draw Opt Complexity: Draw a wide stream of data funneling into a few distinct buckets, which then drop onto a sorted conveyor belt.
Mem Vis (BF): Draw numerous temporary tables generated by correlated subqueries.
Mem Vis (Opt): Draw a Hash Map grouping `user_id` directly to a single `total_spent` integer.
