Problem Title: Numbers At Most N Given Digit Set
Brute Force Method: Generate every possible combination of digits from the set. For each combination, check if the resulting number is <= N.
BF Complexity Visuals: Permutation Search. $O(\text{Digits}^\text{length of} N)$.
How to Draw BF: Draw a tree where each node has D branches (one for each digit). The tree grows to the depth of the number of digits in N.
Optimised Pattern: Digit DP (Combinatorial Counting).
Opt Vis. Technique: The "Slot-by-Slot" Multiplier.
Dry Run (Opt): 1. Count all numbers with fewer digits than N (D^1 + D^2...). 2. For numbers with the same digits as N, iterate from left to right. For each position, count digits in the set smaller than N[i]. If N[i] is in the set, move to the next position; otherwise, stop.
How to Draw (Opt): Draw N = 564. Slots: [__] [__] [__]. 1. Fill 1 and 2 slot options. 2. For 3 slots, if digit set is {1, 3, 5}, the first slot can be 1 or 3 (2 choices x 3 x 3). If first is 5, look at the second slot.
Opt Complexity Visuals: Logarithmic (Length of N). $O(\log N)$.
How to Draw Opt Complexity: Draw a small timeline with length equal to the number of digits in N (usually <= 10).
Mem Vis (BF): List of all valid numbers: Draw a massive memory block.
Mem Vis (Opt): $O(1)$ Space: Draw a few integer variables for counts and math.
