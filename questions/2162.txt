Problem Title: Minimum Cost to Set Cooking Time
Brute Force Method: Try every single possible combination of buttons (00:00 to 99:99) and check if they sum to the target seconds.
BF Complexity Visuals: Exhaustive list of timestamp strings.
How to Draw BF: Draw a clock. Write out "99 seconds" as "01:39" and "00:99". Calculate costs for both. Write $O(1)$ (Fixed Search).
Optimised Pattern: Greedy (Two-Candidate Check).
Opt Vis. Technique: Comparing two specific "paths" on a numeric keypad.
Dry Run (Opt): Convert target seconds to `mins:secs`. Candidate 1: `mins` and `secs` (if `secs &lt; 100`). Candidate 2: `mins-1` and `secs+60` (if `mins &gt; 0`). Calculate push/move cost for both.
How to Draw (Opt): Draw a 3x3 phone keypad. Trace the finger movement for "1-3-9" vs "9-9". Mark each button press and finger slide with a cost.
Opt Complexity Visuals: $O(1)$ (Only 2 cases to check).
How to Draw Opt Complexity: Two dots representing the only two logical possibilities.
Mem Vis (BF): No significant memory used.
Mem Vis (Opt): $O(1)$ primitive variables for costs.
