Problem Title: Check if Number is a Sum of Powers of Three
Brute Force Method: Recursive backtracking: try including every power of 3 (3^0, 3^1, 3^2...) up to n to see if any subset sums to n.
BF Complexity Visuals: Binary Decision Tree.
How to Draw BF: Draw a root. Branch "Include" or "Exclude" for each power of 3. $O(2^{\log_3 n})$.
Optimised Pattern: Base 3 Representation (Ternary Logic).
Opt Vis. Technique: The "Digit Filter".
Dry Run (Opt): Convert n to base 3. If any digit in the base-3 representation is '2', it means you need to use a power of 3 twice, which is not allowed. If only '0's and '1's, return True.
How to Draw (Opt): Perform repeated division by 3. If `n % 3 == 2`, write "FAIL". Otherwise, keep dividing.
Opt Complexity Visuals: Logarithmic Reduction.
How to Draw Opt Complexity: Draw a number shrinking by 1/3 each time. Label $O(\log_3 n)$.
Mem Vis (BF): Recursion stack.
Mem Vis (Opt): A single integer variable. $O(1)$ space.
