Problem Title: Robot Bounded In Circle
Brute Force Method: Simulate the robot's movement for thousands of iterations (e.g., 10,000) and check if it ever returns to (0,0).
BF Complexity Visuals: A long, meandering path that may or may not eventually loop back.
How to Draw BF: Draw (0,0). Trace "GLGLG...". Keep tracing until you run out of paper or see a loop. Write $O(\text{Infinity}/\text{Heuristic})$.
Optimised Pattern: Vector Math / State Check.
Opt Vis. Technique: A "Compass Reset". If the robot isn't facing North after one cycle, it's guaranteed to loop eventually.
Dry Run (Opt): Execute the instructions once. If the robot is back at (0,0) OR it is facing South, East, or West, return true. If it moved away and still faces North, it's gone forever.
How to Draw (Opt): Draw a small coordinate plane. Mark (0,0). Trace one cycle. Draw an arrow for the final position and a small compass for the final direction.
Opt Complexity Visuals: A single pass through the instruction string.
How to Draw Opt Complexity: Draw the instruction string with one arrow. Write $O(N)$ Time.
Mem Vis (BF): Infinite storage of path coordinates to detect cycles.
Mem Vis (Opt): Four integer variables: x, y, dx, dy. $O(1)$ space.
