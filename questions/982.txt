ID: 982
Problem Title: Triplets with Bitwise AND Equal To Zero
Brute Force Method: Three nested loops checking every possible combination `(i, j, k)` to see if `(A[i] &amp; A[j] &amp; A[k]) == 0`.
BF Complexity Visuals: $O(N^3)$. For N=1000, this is 1 billion operations.
How to Draw BF: Draw a 3D cube of points. Every point in the volume must be checked manually.
Optimised Pattern: Precomputed Pairwise Frequency (Hashmap)
Opt Vis. Technique: "The Two-Step Filter." First, calculate the AND result for every *pair* and store the frequency. Then, iterate the array one more time to see which pairs "cancel out" the third element.
Dry Run (Opt): 1. `count[A[i] &amp; A[j]]++` for all pairs. 2. For each `x` in `A`: iterate through `count` map; if `(x &amp; pair_and) == 0`, `total += count[pair_and]`.
How to Draw (Opt): Draw two arrays A and A. Show them "merging" into a frequency table of AND results. Then show a third array A "pinging" that table.
Opt Complexity Visuals: $O(N^2 + N \cdot 2^{16})$. The second step is bounded by the max possible bitwise value (65536), not N.
How to Draw Opt Complexity: Draw a 2D grid for the N^2 part, followed by a linear scan of a 65,536-entry array.
Mem Vis (BF): $O(1)$ extra space, but unusable time.
Mem Vis (Opt): $O(2^{16})$ space for the frequency map. Draw a single row of 65,536 boxes.
