ID: 69
Problem Title: Sqrt(x)
Brute Force Method: Linear Search Start from 1, 2, 3... and square each number until i x i &gt; x. The answer is i-1.
BF Complexity Visuals: The Incrementing Bar
How to Draw BF: Draw a number line. Draw a pointer starting at 1 and stepping right one unit at a time. The distance traveled can be up to √x. Time: $O(√x)$.
Optimised Pattern: Binary Search (Range [0, x])
Opt Vis. Technique: The Half-Way Cut
Dry Run (Opt): Search in the range [0, x]. If mid x mid &gt; x, search the left half. If mid x mid <= x, record mid and search the right half to find a larger potential root.
How to Draw (Opt): Draw a line from 0 to 100. Mark 50. 50^2 = 2500 (Too big). Cut range to [0, 49]. Mark 25. 25^2 = 625 (Too big). Repeat until range collapses.
Opt Complexity Visuals: Logarithmic Halving
How to Draw Opt Complexity: Draw a block representing x. Draw a line cutting it in half, then half again. Total steps = log(x). Time: $O(\log x)$.
Mem Vis (BF): $O(1)$ space.
Mem Vis (Opt): Draw exactly three variables: `low`, `high`, `mid`. $O(1)$ space.
