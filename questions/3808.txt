Problem Title: Find Emotionally Consistent Users (Med.)
Brute Force Method: SQL/Pandas: Self-joins for every day/interaction to check if sentiments match over a rolling chronological window.
BF Complexity Visuals: Cartesian product (NxN) table explosion showing overlapping date checks.
How to Draw BF: Draw Table A, draw Table B. Draw lines connecting EVERY row to EVERY other row to filter dates.
Optimised Pattern: SQL Window Functions (LAG/LEAD) / Pandas `groupby().rolling()`.
Opt Vis. Technique: A sliding window overlay on a chronologically sorted table partitioned by User ID.
Dry Run (Opt): Sort by User, Date. Add a column `prev_sentiment` using LAG(). Compare current to previous. If streak hits target, flag user.
How to Draw (Opt): Draw a table (User, Date, Emotion). Add a new column next to it. Draw an arrow from row N-1 Emotion to row N New Column.
Opt Complexity Visuals: Single pass sorting and scanning pipeline.
How to Draw Opt Complexity: Draw a funnel for "Sort $O(N \log N)$", then a straight pipe for "Window Scan $O(N)$".
Mem Vis (BF): Temporary tempdb storage blowing up due to quadratic intermediate tables generated by self-joins.
Mem Vis (Opt): In-memory partition blocks holding only the current user's window frame $O(\text{Window Size})$.
