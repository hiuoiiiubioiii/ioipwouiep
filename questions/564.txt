Problem Title: Find the Closest Palindrome
Brute Force Method: Iterate upward and downward from the given number one by one until a palindrome is found.
BF Complexity Visuals: Linear search in a space that could be millions of numbers wide. $O(N)$.
How to Draw BF: Draw a number line. Start at N and move left and right step-by-step, checking each number for symmetry.
Optimised Pattern: Prefix Manipulation + Candidate Testing.
Opt Vis. Technique: A "mirror" at the center. You take the first half of the number and reflect it. You also test the reflect of (prefix + 1) and (prefix - 1).
Dry Run (Opt): 1. Take first half as `prefix`. 2. Candidates: Reflected prefix, reflected `prefix+1`, reflected `prefix-1`, `10^(n-1)-1`, and `10^n+1`. 3. Pick the closest one.
How to Draw (Opt): Write the number: `12345`. Take `123`. Mirror it: `12321`. Also try `124` mirrored: `12421` and `122` mirrored: `12221`. Compare distances.
Opt Complexity Visuals: Constant number of string operations. $O(\log N)$ (number of digits).
How to Draw Opt Complexity: Draw a single box with 5 candidate paths branching out.
Mem Vis (BF): Large string objects created during the linear search.
Mem Vis (Opt): A list of 5 candidate strings. $O(\log N)$ space.
