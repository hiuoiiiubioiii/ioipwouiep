Problem Title: Second Highest Salary
Brute Force Method: Sort the unique salaries in descending order and pick the second one. Fails on edge cases where only one salary exists.
BF Complexity Visuals: A "Sorting Filter" diagram where N items are sorted, taking $O(N \log N)$ time.
How to Draw BF: Draw a list of random numbers. Draw a sorted list. Circle the 2nd item. Draw a question mark if the list only has 1 item.
Optimised Pattern: Subquery with LIMIT 1 OFFSET 1 wrapped in IFNULL.
Opt Vis. Technique: A "Box within a Box." The inner box finds the 2nd value, the outer box ensures a NULL return if the inner one is empty.
Dry Run (Opt): Select max salary. Select max salary again where salary is less than the first max. This handles the "second highest" logic perfectly.
How to Draw (Opt): Draw Table A. Draw a smaller Table B (filtered for salaries &lt; Max). Circle the highest value in Table B.
Opt Complexity Visuals: $O(N)$ with a single scan if using `MAX()` twice, or index-based $O(\log N)$.
How to Draw Opt Complexity: Draw an Index Tree (B-Tree). Show the pointer jumping directly to the second-to-last leaf node.
Mem Vis (BF): Visualise a full copy of the table being sorted in temp space (O(N)).
Mem Vis (Opt): Zero extra space beyond the pointer to the result (O(1)).
