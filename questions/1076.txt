Problem Title: Project Employees II
Brute Force Method: (SQL) Group by project, count employees, sort descending, and take the top. Use another query to check if others have the same count.
BF Complexity Visuals: Multiple independent sorting and filtering passes.
How to Draw BF: P1: 3, P2: 3, P3: 2. Sort: 3, 3, 2. Max is 3. Find all projects with 3. Write $O(\text{Projects} \\text{log Projects})$.
Optimised Pattern: Group By + Having (Subquery Max).
Opt Vis. Technique: A "High-Water Mark". Count everyone, find the highest count, and keep everyone who reached that peak.
Dry Run (Opt): `SELECT project_id FROM Project GROUP BY project_id HAVING COUNT(employee_id) = (SELECT COUNT(employee_id) FROM Project GROUP BY project_id ORDER BY COUNT(employee_id) DESC LIMIT 1)`.
How to Draw (Opt): Draw a bar chart of employee counts per project. Draw a horizontal line at the level of the tallest bar. List all bars touching the line.
Opt Complexity Visuals: One pass for counts, one pass for the max, one pass for filter.
How to Draw Opt Complexity: Draw the table collapsing into counts, then a filter gate. Write Time: $O(\text{Projects})$.
Mem Vis (BF): Multiple temporary result sets for sorting.
Mem Vis (Opt): A small internal hash table for counts. $O(\text{UniqueProjects})$ space.
