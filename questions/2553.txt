ID: 2553
Problem Title: Separate the Digits in an Array
Brute Force Method: Convert the whole array to a single string, then iterate through the string to create a new integer array.
BF Complexity Visuals: A string concatenation pipeline that creates massive temporary objects in memory.
How to Draw BF: Write [13, 25]. Write "1325". Then write [1, 3, 2, 5].
Optimised Pattern: Iterative Digit Extraction (Modulo &amp; Stack).
Opt Vis. Technique: A "Digit Peeler" where each number is broken down from right to left, then reversed or appended to a result list.
Dry Run (Opt): For each number in the array, extract digits using % 10. Since this gives digits in reverse (e.g., 13 -&gt; 3, 1), store them in a temporary stack or list and reverse them, then append to the final answer.
How to Draw (Opt): Draw the array. Under each number, draw a small vertical list of its digits. Then read the entire page left-to-right, top-to-bottom.
Opt Complexity Visuals: $O(\text{Total Number of Digits})$, which is effectively $O(N)$ given constraints.
How to Draw Opt Complexity: Draw an arrow moving through the array. At each stop, draw a "mini-loop" that spins until the number is 0.
Mem Vis (BF): $O(D)$ where D is the total string length of all numbers.
Mem Vis (Opt): $O(N)$ to store the final result list.
