ID: 847
Problem Title: Shortest Path Visiting All Nodes
Brute Force Method: Generate every permutation of nodes and calculate the shortest path to traverse them in that specific order.
BF Complexity Visuals: Traveling Salesperson Explosion. $O(N!)$.
How to Draw BF: Draw N nodes. Draw paths for [1,2,3,4], then [1,2,4,3], then [1,3,2,4]... The list of permutations becomes a massive, dense block of text.
Optimised Pattern: BFS on (CurrentNode, VisitedMask) State Space.
Opt Vis. Technique: Multi-Source Ripple in 2D State Space.
Dry Run (Opt): Use a bitmask to represent visited nodes. State = (node, mask). Start BFS from all nodes simultaneously with their initial masks. The first state to reach mask `(1 &lt;&lt; N) - 1` is the shortest path.
How to Draw (Opt): Draw a grid where rows are Nodes and columns are Bitmasks (0-2^N). Draw arrows showing transitions between "states" as you move and update masks.
Opt Complexity Visuals: Capped State Space. $O(N \\text{times} 2^N)$.
How to Draw Opt Complexity: Draw a 2D matrix of size N \times 2^N. Each cell is visited exactly once in the BFS.
Mem Vis (BF): Massive permutation list: Draw a stack N! levels high.
Mem Vis (Opt): Visited 2D Array / Set: Draw a grid of size 12 \times 4096 (for max N=12).
