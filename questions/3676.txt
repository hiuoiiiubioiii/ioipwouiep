Problem Title: Count Bowl Subarrays (Med)
Brute Force Method: Generate every possible subarray (O(N^2)) and for each, check if the elements decrease then increase (the "bowl" shape) in $O(N)$. Total $O(N^3)$.
BF Complexity Visuals: Nested Array Scan.
How to Draw BF: Draw an array. Draw a bracket for every possible [L, R]. Inside, check if numbers go down then up. With N=10,000, this is 100 billion checks.
Optimised Pattern: Precompute "Longest Decreasing" from left and "Longest Increasing" from right for every index.
Opt Vis. Technique: The "V-Shape" Intersection.
Dry Run (Opt): Think of every index as the potential "bottom" of a bowl. A bowl at index i can extend left as far as numbers are increasing towards the left, and right as far as they increase towards the right.
How to Draw (Opt): Draw an array. For each index, write two numbers: `L` (how many smaller items to the left) and `R` (how many smaller items to the right). Total bowls = sum of `L * R` for all i.
Opt Complexity Visuals: Linear Precomputation Pass.
How to Draw Opt Complexity: Draw one arrow for Left-pass (O(N)), one for Right-pass (O(N)), and one for the final Sum (O(N)). Total $O(N)$.
Mem Vis (BF): Allocating N^2 sub-arrays in memory for validation.
Mem Vis (Opt): Two 1D arrays of size N. Space $O(N)$.
