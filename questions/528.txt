Problem Title: Random Pick with Weight
Brute Force Method: Create a literal array where index `i` is duplicated `weight[i]` times, then pick a random index from it. (Memory Limit Exceeded).
BF Complexity Visuals: A giant pie chart where a weight of 10,000 creates 10,000 individual tiny slices. $O(\text{Sum of weights})$.
How to Draw BF: Draw a massive array filling up a page, writing "0" a hundred times, "1" fifty times, etc.
Optimised Pattern: Prefix Sum Array + Binary Search.
Opt Vis. Technique: A number line divided into segments of unequal lengths. A random dart lands on the line, and you quickly find which segment it hit.
Dry Run (Opt): Init: Build prefix sum array. Pick: Generate random number `1` to `total_sum`. Binary search prefix array to find first value ≥ random number.
How to Draw (Opt): Draw array [1, 3, 4]. Draw prefix sum: [1, 4, 8]. Write Random=6. Draw binary search arrows narrowing down to index 2 (value 8).
Opt Complexity Visuals: Logarithmic halving of the prefix sum array. $O(N)$ Init, $O(\log N)$ Pick.
How to Draw Opt Complexity: Draw the prefix array, cut it in half, cut in half again, pointing to the target.
Mem Vis (BF): An array of size equal to the total sum of all weights combined.
Mem Vis (Opt): A single array of size N storing the prefix sums. $O(N)$ space.
