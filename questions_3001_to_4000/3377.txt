ID: 3377
Problem Title: Digit Operations to Make Two Integers Equal
Brute Force Method: Try every possible single-digit change recursively to find the path from n to m, avoiding primes.
BF Complexity Visuals: Exhaustive State Graph
How to Draw BF: Draw n in the center. Draw lines to all neighbors (changing 1 digit). Repeat for neighbors. $O(10^D)$.
Optimised Pattern: Dijkstra’s Algorithm + Prime Sieve
Opt Vis. Technique: Expanding Cost Wavefront
Dry Run (Opt): Precompute primes using a Sieve. Use a Min-Heap to explore integers with the lowest cumulative "cost" (the value of the number).
How to Draw (Opt): Draw n as a dot. Draw a "circle" around it of numbers reachable by 1 step. Color prime dots red (dead ends). Write the "value" inside each dot.
Opt Complexity Visuals: Priority Queue Expansion
How to Draw Opt Complexity: Draw a 1D number line 1 dots 10000. Mark jumps between dots. The shortest path is the one with the lowest sum of labels. $O(V \log V + E)$.
Mem Vis (BF): A list of all possible paths explored, leading to massive memory usage.
Mem Vis (Opt): A `dist` array of size 10,000 and a Min-Heap. $O(\text{Range})$.
