Problem Title: Maximum Score After Binary Swaps (Med)
Brute Force Method: Generate all possible strings reachable by any number of adjacent swaps of '0' and '1', then calculate the score for each. $O(2^N)$.
BF Complexity Visuals: State Space Reachability Graph.
How to Draw BF: Draw the binary string. Draw branches for every possible swap. Follow the branches to see all unique string outcomes and calculate scores.
Optimised Pattern: Greedy Counting / Frequency Analysis
Opt Vis. Technique: Bit Re-distribution
Dry Run (Opt): Adjacent swaps allow you to move any '1' to any position. To maximize a score (usually based on prefix/suffix or value), count the total number of '1's and '0's and place all '1's at the most "valuable" positions (e.g., the front).
How to Draw (Opt): Write the string. Count the '1's. Draw a new string of the same length, filling the first N slots with '1's and the rest with '0's. This is your "perfect" swap result.
Opt Complexity Visuals: Single Linear Pass $O(N)$.
How to Draw Opt Complexity: Draw a flat timeline with two counters (One for '0', one for '1').
Mem Vis (BF): Recursive state tree storage.
Mem Vis (Opt): $O(1)$ extra space (just bit counters).
