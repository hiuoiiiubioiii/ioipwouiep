Problem Title: Distinct Points Reachable After Substring Removal (Med)
Brute Force Method: Try removing every possible substring [i, j], calculate the final path from the remaining string, and store the endpoint in a Set. $O(N^3)$.
BF Complexity Visuals: Subsegment Deletion Exhaustion.
How to Draw BF: Draw a string "URDL". Remove "U", path "RDL" -&gt; (1, -1). Remove "UR", path "DL" -&gt; (0, -2). With N=2000, this is 8 billion coordinates.
Optimised Pattern: Prefix Sums of Coordinates. Calculate (x, y) for all prefixes and suffixes. Endpoint = Prefix[i-1] + (Total - Prefix[j]).
Opt Vis. Technique: The "Vector Bridge" Calculation.
Dry Run (Opt): Imagine a path as a chain. Removing a link doesn't require rebuilding the whole chain; just snap the two remaining ends together using simple vector addition.
How to Draw (Opt): Precompute pref[i] as the (x, y) after i steps. For a removal of [i, j], the new endpoint is pref[i-1] + (pref[N] - pref[j]). Put these in a Set.
Opt Complexity Visuals: N^2 Constant-Time Lookups.
How to Draw Opt Complexity: Draw an $O(N)$ pass to build prefixes. Then draw a nested $O(N^2)$ loop where each step is a simple subtraction. Total $O(N^2)$.
Mem Vis (BF): Calculating the path from scratch for every single substring removal.
Mem Vis (Opt): A Hash Set of Coordinate pairs and a 1D array of prefix vectors. Space $O(N^2)$ for results.
