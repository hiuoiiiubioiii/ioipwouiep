Problem Title: Ugly Number II
Brute Force Method: Check every integer 1, 2, 3... using the Q263 method until the n^{th} ugly number is found.
BF Complexity Visuals: Linear Scan with Log Check: $O(M \cdot \log M)$ where M is the n^{th} ugly number.
How to Draw BF: Draw a long number line. Put a mark on "1, 2, 3, 4, 5, 6, 8...". Show the gap between marks getting wider and wider.
Optimised Pattern: Three Pointers (Dynamic Programming)
Opt Vis. Technique: Parallel Multiplier Tapes
Dry Run (Opt): Keep three pointers (p2, p3, p5). Next ugly number is min(ugly[p2]*2, ugly[p3]*3, ugly[p5]*5). Increment the pointer(s) that produced the min.
How to Draw (Opt): Draw an array `U`. Place 3 fingers labeled "x2", "x3", and "x5". Move them forward independently like a race to generate the next value.
Opt Complexity Visuals: Single Pass: $O(N)$ Time.
How to Draw Opt Complexity: Draw a straight timeline of length N. Each step involves exactly 3 multiplications and 1 comparison.
Mem Vis (BF): N/A.
Mem Vis (Opt): DP Array: Draw a 1D array of size N that stores previously computed ugly numbers.
