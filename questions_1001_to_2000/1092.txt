ID: 1092
Problem Title: Shortest Common Supersequence
Brute Force Method: Generate every possible string that contains both str1 and str2 as subsequences. Find the shortest one among them.
BF Complexity Visuals: An infinite search space of string combinations.
How to Draw BF: "ab", "ac". Try "abc", "acb", "abac"... Write $O(\text{Exponential})$.
Optimised Pattern: Dynamic Programming (LCS Variation).
Opt Vis. Technique: A "Master Blueprint". Find the Longest Common Subsequence (LCS) first, then fill in the unique characters around it.
Dry Run (Opt): Compute the 2D DP table for LCS. Backtrack from `dp[n][m]`. If characters match, add once. If not, add the character that was dropped to reach the current DP state. Reverse result.
How to Draw (Opt): Draw the DP matrix for LCS. Trace the diagonal path from bottom-right to top-left. For horizontal/vertical moves, write down the corresponding character. For diagonals, write the shared character.
Opt Complexity Visuals: Two-pass process (compute matrix, then backtrack).
How to Draw Opt Complexity: Draw a grid with a single snaking path from end to start. Write Time: $O(N \cdot M)$.
Mem Vis (BF): Massive string allocations.
Mem Vis (Opt): $O(N \cdot M)$ space for the DP table.
